Question: How should we handle the blog/thoughts page?

We need to decide on the architecture for displaying your Facebook posts on the Thoughts & Stuff page. Each option has different trade-offs around build time, interactivity, and maintenance.

Option A: Static Build

Pre-render all posts at build time. Fast page loads, no client-side data fetching. Posts baked into the JS bundle.

✓ Fastest load · ✓ Works offline · ✓ Simple
✗ Large bundle · ✗ Rebuild to add posts

Option B: API Route + Client Fetch

Posts served from a Next.js API route reading JSON on the server. Client fetches pages on scroll (infinite scroll).

✓ Small initial load · ✓ Easy to update
✗ Needs JS · ✗ Slight delay on scroll

Option C: Hybrid ISR

First page of posts statically rendered, subsequent pages fetched on demand with Incremental Static Regeneration.

✓ Best of both · ✓ SEO friendly
✗ More complex · ✗ ISR config needed

Option D: Static HTML Export

Generate a standalone HTML page (like the current mockup) and serve it from /public. No React hydration needed.

✓ Zero JS · ✓ Maximum speed · ✓ Portable
✗ No interactivity · ✗ Manual updates
